# MSW 용어 및 핵심 개념 정리 (MSW Terminology)

메이플스토리 월드(MSW) 개발에 필수적인 핵심 개념과 용어를 정리한 지침입니다. 에이전트는 이 스킬을 통해 MSW 플랫폼의 독자적인 구조를 정확히 이해하고 일관된 용어를 사용하여 소통 및 구현을 진행해야 합니다.

## 1. 엔티티 시스템 (Entity System)

MSW는 엔티티-컴포넌트-프로퍼티(ECP) 구조를 기반으로 합니다.

- **엔티티 (Entity)**: 월드 내에 존재하는 모든 객체입니다. 그 자체로는 빈 껍데기이며, 여러 컴포넌트의 집합체로 기능합니다.
- **컴포넌트 (Component)**: 엔티티의 특정 기능을 담당하는 독립적인 단위입니다. (예: `TransformComponent`, `SpriteRendererComponent`) 컴포넌트를 조합하여 엔티티의 성격과 동작을 결정합니다.
- **프로퍼티 (Property)**: 컴포넌트 내부의 세부 설정 값입니다. 같은 컴포넌트를 사용하더라도 프로퍼티 값을 다르게 설정하여 엔티티마다 개별적인 특성을 부여할 수 있습니다.

## 2. 실행 환경 및 네트워크 (Execution & Network)

- **실행 공간 (Execution Space)**: 로직이 실제로 실행되는 위치로 **서버(Server)** 또는 **클라이언트(Client)**로 구분됩니다.
- **실행 공간 제어 (Execution Space Setting)**: 특정 함수가 서버 혹은 클라이언트 중 어디에서 실행될지 설정하는 기능입니다.
- **동기화 (Synchronization)**: 서버와 클라이언트 간의 데이터를 일치시키는 과정입니다. 주로 프로퍼티 동기화를 통해 한쪽에서 변경된 값이 다른 쪽에도 반영되도록 합니다.
- **임베디드 서버 (Embedded Server)**: 제작 중 테스트를 위해 로컬 환경에서 구동되는 가상 서버입니다.

## 3. 계층 구조 및 모델 (Hierarchy & Model)

- **계층 구조 (Hierarchy)**: 월드 내 엔티티들의 부모-자식 관계를 나타내는 트리 구조입니다. `World`, `common`, `maps`, `ui` 영역으로 나뉩니다.
- **모델 (Model)**: 컴포넌트와 프로퍼티 설정이 완료된 엔티티의 원형(Original)입니다. 이를 복제(Instantiate)하여 실제 월드에 배치합니다. `NativeModel`과 크리에이터가 직접 만든 모델이 있습니다.
- **RUID (Resource Unique ID)**: 이미지, 사운드 등 개별 리소스가 가지는 고유한 식별자입니다.

## 4. 특수 스크립트 타입 (Special Script Types)

MSW는 일반적인 로직 외에 특정 시스템과 연계된 전용 타입을 제공합니다.

- **EventType**: 사용자 정의 이벤트를 정의할 때 사용하며, 엔티티 이벤트 시스템과 연동됩니다.
- **ItemType**: 아이템의 성격과 데이터를 정의합니다. `ItemService`와 연동되며 `OnCreate` 이벤트를 포함합니다.
- **BTNodeType**: 행동 트리(Behaviour Tree) 기반 AI를 구현할 때 사용하는 노드 타입입니다. `OnBehave` 함수를 필수로 구현해야 합니다.
- **StateType**: 상태 머신(FSM)을 구현하기 위한 타입입니다. `OnEnter`, `OnUpdate`, `OnExit` 등을 통해 상태 전이를 관리합니다.
- **StructType**: 여러 데이터를 하나로 묶어 관리하는 사용자 정의 데이터 구조체입니다.

## 5. 워크스페이스 구조 (Workspace)

- **MyDesk**: 크리에이터가 생성한 스크립트, 모델, 리소스가 저장되는 개인 작업 공간입니다.
- **BaseEnvironment**: MSW에서 기본 제공하는 `NativeScripts`, `NativeModel` 등이 포함된 읽기 전용 환경입니다.
- **DefaultPlayer**: 워크스페이스에 포함되어 있으며, 플레이어의 기초가 되는 엔티티 모델입니다.

## 언제 이 스킬을 사용하는가

- MSW의 용어 정의가 모호하여 정확한 의미 확인이 필요할 때
- 엔티티와 컴포넌트의 관계를 재설계하거나 구조를 파악할 때
- 서버/클라이언트 간의 실행 로직을 어떻게 구분해야 할지 판단할 때
- `ItemType`이나 `BTNodeType` 등 전용 타입의 기초 구조와 이벤트 함수가 궁금할 때

## 사용 방법

1. 구현하려는 기능이 MSW의 어떤 핵심 개념(엔티티, 타입, 실행 공간 등)에 해당하는지 본 문서를 통해 확인합니다.
2. 명명 규칙이나 시스템 설계 시 본 문서에 명기된 표준 용어를 준수합니다.
3. 복잡한 로직 설계 시 서버/클라이언트의 역할 분담을 본 문서의 '실행 공간' 개념에 비추어 검토합니다.
