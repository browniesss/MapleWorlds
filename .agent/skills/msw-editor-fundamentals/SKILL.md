# MSW Editor Fundamentals & Best Practices

이 스킬은 메이플스토리 월드(MSW) 에디터의 독특한 아키텍처와 개발 시 반드시 고려해야 할 핵심 특징들을 안내합니다.

## 언제 이 스킬을 사용하는가 (When to use this skill)

- MSW 에디터 내에서 엔티티 계층 구조를 설계하거나 컴포넌트를 구현할 때.
- 서버와 클라이언트 간의 동기화 방식이 헷갈리거나 실행 제어 로직을 작성할 때.
- 월드, 스크린, UI 좌표계 간의 변환 또는 해상도 대응이 필요할 때.
- 스크립트의 실행 순서(라이프사이클)에 따른 로직 배치가 필요할 때.

## 사용 방법 (How to use)

### 1. 엔티티 및 컴포넌트 시스템 (E-C-P)
- **객체 지향 구조**: 엔티티는 컴포넌트의 집합체이며, 각 컴포넌트는 독립적인 기능을 수행합니다.
- **재사용성**: 기능을 세분화하여 컴포넌트로 만들고, 이를 다양한 엔티티에 조합하여 재사용하십시오.

### 2. 네트워크 및 로컬화된 엔티티
- **Localized Entity**: 클라이언트에서만 존재하는 엔티티(UI, 이펙트용 객체 등)입니다.
    - **주의**: 로컬 엔티티의 컴포넌트에서는 `[Server]`, `[ServerOnly]` 함수를 호출할 수 없으며, 서버 검증이 불가능합니다.
- **동기화 방향**: 프로퍼티 동기화는 항상 **Server -> Client** 방향으로만 일어납니다. 클라이언트에서의 값 변경은 서버로 전달되지 않습니다.

### 3. 세 가지 좌표계의 이해
- **World 좌표**: `TransformComponent`가 사용하는 맵 기반 좌표 (중앙이 원점).
- **Screen 좌표**: 실제 기기의 물리적 픽셀 좌표 (좌측 하단 원점). 터치 이벤트 시 활용.
- **UI 좌표**: 해상도에 독립적인 1920x1080 기준 좌표 (중앙 원점).
    - **Best Practice**: `UITransformComponent`의 **Anchor Presets**를 활용하여 다양한 해상도(모바일, PC)에 대응하십시오.

### 4. 스크립트 라이프사이클 (Lifecycle)
순차적으로 발생하는 이벤트의 특징을 이해하십시오:
1. `OnInitialize`: 엔티티 생성 시 최우선 호출.
2. `OnMapEnter`: 플레이어가 맵에 진입할 때마다 호출.
3. `OnBeginPlay`: 초기화가 완료된 후 게임 로직 시작 시 호출. (가장 많이 사용)
4. `OnUpdate`: 매 프레임 호출.

### 5. 월드 인스턴스 및 데이터 공유
- **공유 데이터**: 모든 월드 인스턴스는 동일한 `DataStorage`를 공유합니다. 랭킹이나 유저 정보 처리에 유의하십시오.
- **독립 실행**: 각 인스턴스는 독립적인 서버 메모리를 가지므로, 인스턴스 간 직접적인 메모리 공유는 불가능합니다.

### 6. 사용자 정의 타입 (Custom Types)
- **구성**: `EventType`, `ItemType`, `StructType` 등 용도에 맞는 타입을 정의하여 관리하십시오.
- **제약**: 타입은 컴포넌트가 아니므로 엔티티에 부착할 수 없으며, 실행 제어 가용성이 없습니다.

## 의사 결정 가이드 (Decision Tree)

1. **이 엔티티가 서버에서 관리되어야 하는가?**
    - Yes -> 일반 엔티티.
    - No (UI/장식) -> **Localized Entity** 설정으로 서버 부하 감소.
2. **UI 요소를 배치하는가?**
    - Yes -> 절대 좌표 대신 **Anchor Presets**를 사용하여 해상도 대응 여부 확인.
3. **클라이언트에서 서버 함수가 불리지 않는가?**
    - Yes -> 해당 엔티티가 **Localized Entity**인지 확인.
4. **모든 유저에게 공통된 데이터를 보여줘야 하는가?**
    - Yes -> `GlobalDataStorage` 활용 (모든 인스턴스 공용).
5. **라이프사이클 단계 선택?**
    - 전역 초기 설정 -> `OnInitialize`.
    - 게임 플레이 로직 시작 -> `OnBeginPlay`.
    - 맵 이동 시마다 초기화 -> `OnMapEnter`.
