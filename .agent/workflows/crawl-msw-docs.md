> [!IMPORTANT]
> 현재 시스템의 Playwright 브라우저 연결 문제로 인해 `open_browser_url` 등 브라우저 기반 도구 사용이 불가능합니다.
> 모든 크롤링 작업은 **`read_url_content` 도구**를 사용한 텍스트 추출 방식으로 진행하십시오.

1. 사용자로부터 크롤링을 시작할 **루트 URL**을 입력받습니다.
2. 프로젝트 루트에 `/MSW_DOC` 디렉토리가 없다면 생성합니다.
3. 다음 규칙에 따라 재귀적으로 문서를 수집합니다:
    - **도구 선택**: 문서 수집 시 브라우저 도구 대신 반드시 **`read_url_content`**를 사용합니다. HTML이 마크다운으로 자동 변환되어 제공됩니다.
    - **깊이 제한**: 루트 URL로부터 최대 **10단계(Depth 10)**까지만 탐색합니다.
    - **레이트 리밋**: 각 문서를 읽은 후, 다음 문서를 읽기 전 반드시 **5초간 대기**합니다.
        - // turbo
        - `run_command`를 사용하여 `powershell -Command "Start-Sleep -Seconds 5"`를 실행하세요.
    - **중복 제거**: 이미 있는 문서라면 스킵합니다.
    - **콘텐츠 최적화**: `read_url_content`로 수집된 마크다운 내용에서 Antigravity AI(수석 개발자)가 참조하기 좋게 다음 구조를 유지하며 리팩토링합니다:
        - 핵심 요약 (Key Takeaways)
        - 상세 API 사양 또는 기술 설명
        - 예제 코드 (mLua 스타일 준수)
        - 관련 링크 및 참조
    - **파일 저장**: 각 페이지는 `/MSW_DOC` 아래에 고유한 파일명을 가진 `.md` 파일로 저장합니다.
4. **오류 처리**: 만약 `read_url_content`로도 내용을 가져오지 못하거나 접근이 차단된 경우, 해당 URL을 기록하고 건너뛴 뒤 사용자에게 최종 보고 시 알립니다.
5. 모든 작업이 완료되면 수집된 문서의 총 개수, 건너뛴 URL, 그리고 저장된 파일 목록을 사용자에게 보고합니다.
